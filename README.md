# Django + Inertia.js + React Multi-Tenant App

A modern multi-tenant SaaS application using Django backend with Inertia.js and React frontend. Each business gets its own isolated PostgreSQL schema for complete data separation.

рҹҡҖ **[Quick Start Guide вҶ’](NEXT_STEPS.md)** | рҹ“– **[Full Setup Guide вҶ’](TENANCY_SETUP.md)** | рҹ’Ў **[Best Practices вҶ’](TENANCY_BEST_PRACTICES.md)**

## Tech Stack

- **Backend**: Django 5.0 (LTS)
- **Multi-Tenancy**: django-tenants (schema-based)
- **Frontend**: React 18 + Inertia.js
- **Bundler**: Vite
- **Styling**: Tailwind CSS
- **Database**: PostgreSQL (required for schema support)
- **Deployment**: Docker + Nginx + Gunicorn

## Multi-Tenancy Architecture

This application uses **schema-based multi-tenancy**:
- вң… Each business/tenant gets its own PostgreSQL schema
- вң… Complete data isolation between tenants
- вң… Shared infrastructure (single Django codebase, database)
- вң… Tenant routing via domains/subdomains
- вң… Example: `acme.yourdomain.com` вҶ’ acme schema, `xyz.yourdomain.com` вҶ’ xyz schema

## Features

- вң… **Multi-tenant architecture** with schema-based isolation
- вң… **PostgreSQL schemas** - each tenant has isolated data
- вң… **Domain-based routing** - automatic tenant detection
- вң… Django handles all routing, auth, and business logic
- вң… React for UI components via Inertia.js (no REST API needed)
- вң… Server-side session authentication (no JWT)
- вң… CSRF protection enabled
- вң… Vite HMR in development
- вң… Static files served in production (no Node server)
- вң… Docker multi-stage build for production
- вң… Nginx reverse proxy
- вң… Easy tenant creation via management commands

## Quick Start (Multi-Tenancy)

### Prerequisites

- Python 3.11+
- Node.js 20+
- **PostgreSQL 12+** (Required for schema support)

### 1. Install PostgreSQL

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install postgresql postgresql-contrib

# macOS
brew install postgresql
brew services start postgresql
```

### 2. Run Setup Script

```bash
# Make script executable
chmod +x setup_tenancy.sh

# Run setup (creates database, migrations, public tenant)
./setup_tenancy.sh
```

### 3. Create Your First Business Tenant

```bash
# Create tenant
python manage.py create_tenant \
    --name="ACME Corporation" \
    --schema="acme" \
    --domain="acme.localhost"

# Migrate tenant schema
python manage.py migrate_schemas --schema=acme

# Create admin user for tenant
python manage.py tenant_command createsuperuser --schema=acme
```

### 4. Run Development Server

```bash
# Terminal 1: Django backend
python manage.py runserver

# Terminal 2: Frontend dev server (in frontend/ directory)
cd frontend
npm install
npm run dev
```

### 5. Access Your Application

- **Public site**: http://localhost:8000
- **ACME tenant**: http://acme.localhost:8000
- **Another tenant**: http://xyz.localhost:8000

> **Note**: Modern browsers automatically resolve `*.localhost` to 127.0.0.1

## Development Setup (Manual)

### Prerequisites

- Python 3.11+
- Node.js 20+
- PostgreSQL 12+

### 1. Clone and Setup Environment

```bash
# Create .env file
cat > .env << EOF
DB_NAME=box_db
DB_USER=postgres
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=5432
SECRET_KEY=your-secret-key
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1,.localhost

```

### 2. Setup Python Environment

```bash
# Create virtual environment
python -m venv venv

# Activate virtual environment
# On Linux/Mac:
source venv/bin/activate
# On Windows:
# venv\Scripts\activate

# Install Python dependencies
pip install -r requirements.txt
```

### 3. Setup Database and Tenants

```bash
# Create PostgreSQL database
sudo -u postgres psql -c "CREATE DATABASE box_db;"

# Run shared schema migrations
python manage.py migrate_schemas --shared

# Create public tenant
python manage.py create_tenant \
    --name="Public" \
    --schema="public" \
    --domain="localhost"

# Create your first business tenant
python manage.py create_tenant \
    --name="Demo Business" \
    --schema="demo" \
    --domain="demo.localhost"

# Migrate tenant schema
python manage.py migrate_schemas --schema=demo
```

### 4. Setup Frontend

```bash
cd frontend
npm install
```

## Production Deployment

### Build and Deploy with Docker

```bash
# Build and start all services
docker-compose up -d --build

# Run migrations
docker-compose exec web python manage.py migrate

# Create superuser
docker-compose exec web python manage.py createsuperuser

# View logs
docker-compose logs -f web
```

Visit: **http://localhost**

### Manual Production Build

```bash
# 1. Build frontend assets
cd frontend
npm run build
cd ..

# 2. Collect static files
python manage.py collectstatic --noinput

# 3. Run migrations
python manage.py migrate

# 4. Start Gunicorn
gunicorn config.wsgi:application --bind 0.0.0.0:8000 --workers 4
```

### Docker Services

- **db**: PostgreSQL database
- **web**: Django application (Gunicorn)
- **nginx**: Reverse proxy and static file server

## Key Files Explained

### Django Settings ([config/settings.py](config/settings.py))

- Uses environment variables via `python-decouple`
- Configured for Inertia.js middleware
- WhiteNoise for static file serving
- PostgreSQL database

### Base Template ([templates/base.html](templates/base.html))

- Loads Vite dev server in development
- Loads built assets in production
- CSRF token for Inertia requests
- Single `<div id="app">` mount point

### Vite Config ([frontend/vite.config.js](frontend/vite.config.js))

- Dev server on port 5173
- Builds to `../static/dist/`
- Manifest mode for Django integration

### Views ([app/views.py](app/views.py))

- `@inertia` decorator for GET requests
- Manual `inertia_render` for POST with validation errors
- Server-side validation
- Django sessions for auth

### Nginx ([nginx.conf](nginx.conf))

- Serves static files directly
- Proxies dynamic requests to Django
- Proper headers for security

## How It Works

1. **Django** handles all HTTP requests
2. For Inertia requests, Django returns JSON with page component name and props
3. **Inertia.js** intercepts links and form submissions
4. React components receive props from Django
5. In development: Vite HMR provides instant updates
6. In production: Frontend is pre-built and served as static files

## Troubleshooting

### "Cannot connect to database"
- Ensure PostgreSQL is running
- Check database credentials in `.env`

### "Vite not loading in development"
- Make sure Vite dev server is running on port 5173
- Check `frontend/vite.config.js` configuration

### "Static files not loading in production"
- Run `python manage.py collectstatic`
- Check `STATIC_ROOT` and `STATIC_URL` in settings

### "CSRF verification failed"
- Ensure `{% load static %}` in base template
- Check CSRF token meta tag in base.html
- Verify Inertia middleware is enabled
